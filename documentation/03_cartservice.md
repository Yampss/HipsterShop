# Cart Service

## Overview
The Cart Service is responsible for managing the user's shopping session (adding, retrieving, and emptying items in the cart). We migrated this service from using an in-memory Redis cache to a durable PostgreSQL database.

* **Language:** C# (.NET Core)
* **Port Exposed:** 7070 (gRPC)

## How the Code Works
The core logic resides in `PostgresCartStore.cs`. 
1. The service receives a `userId` (a UUID generated by the Frontend stored in a browser cookie).
2. For an `AddItem` request, it connects to PostgreSQL, reads the existing `cart_data` (which is a serialized Protobuf `BYTEA` blob), deserializes it into C# objects, increments the quantity, reserializes it, and saves it back using an `INSERT ... ON CONFLICT DO UPDATE` (Upsert) SQL query.
3. `Startup.cs` manages dependency injection, choosing to instantiate our `PostgresCartStore` class because the `DB_HOST` environment variable is detected at startup.

## Potential Trainer Questions

**Q: Wait, why did you store the cart data as a `BYTEA` blob instead of individual relational columns?**
**A:** "This is a tactical design choice. Since the `CartItem` schema is defined strictly by our `demo.proto` protobuf file, we simply serialize the strongly-typed Protobuf object directly into bytes and store it. This gives us the durability of a relational Postgres database, while keeping the lightning-fast read/write patterns of a NoSQL document store, without needing complex SQL JOINs to reconstruct the cart."

**Q: How does this service securely connect to the database in Kubernetes?**
**A:** "We use Kubernetes `Secrets`. Inside our `cartservice.yaml` manifest, we use `valueFrom: secretKeyRef` to inject the database password directly into the container's environment variables at runtime, ensuring no passwords are hardcoded in the C# source code."
